# Support

If MAS fails to start or analyze a workspace:

1. Confirm Docker is installed and running.
2. Confirm the configured `masi.repoRoot` points to your MAS repository.
3. Confirm the configured `masi.pythonPath` points to the MAS virtual environment Python executable.
4. Run `MAS: Install Runtime` once before starting the API.
5. Open the `MAS` output channel in VS Code for logs.

For publishing or packaging issues, use the checklist in [PUBLISHING.md](./PUBLISHING.md).
